About Adskate

AdSkate is building cookieless solutions for the privacy-first world. Our technology enables businesses to select ad placements and optimize creative elements that drive successful conversions. Our customer have seen a 15x increase in conversions and a 36% decrease in ad spend.

Identify product requirements

The advertising industry is currently witnessing a surge in the use of AI, with numerous startups venturing into the field. Adskate must act swiftly by introducing its MVP to identify its product-market fit at the earliest. Prioritizing features in a ruthless manner is crucial to expedite the process, as it will enable Adskate to gather valuable feedback from its initial customers and reduce time-to-market. Below are the MVP components I proposed:

  1. AI algorithm: Adskate's advanced AI algorithm uses natural language processing (NLP) and computer vision to analyze campaign data and provide actionable insights.
  2. Advertiser data input: Advertisers can upload their campaign performance logs and ad creative to Adskate.
  3. Ad creative analysis: Advertisers can access detailed analysis of their ad creative's performance.
  4. Campaign optimization recommendations: Adskate provides advertisers with recommendations for optimizing their campaigns based on the analytics results.

System Design

The front-end application allows advertisers to upload their data and access reports. The data will be stored in an object storage like S3 bucket. As soon as the data is uploaded, our algorithm processes it, then generate multiple reports regarding CTA, object, and audience. These reports are stored as JSON objects in RDS relational database along with the links pointing to advertiser-uploaded data in S3 bucket. We serve the reports to customers via an HTML server. When advertisers log in and request a report with their job_id, the API immediately sends a request to the HTML server, which returns the report to the advertiser.

UI/UX Design on Figma, Webflow, and Bubble.io

The report interface

  1. Two dropdowns allow advertisers to slice their data by different criteria, such as geography or campaign type, and to choose the key metric they want to optimize for.
  2. A side navigation panel lets users switch between different views, including CTA analysis, audience insights, and object performance.
  3. The comparison screen displays good and bad creative side by side, along with performance statistics highlighting the differences between them.
  4. Recommendations for optimizing campaigns, as well as an option to download a list of creative assets that need to be improved.

Build web application

  1. Registration and login (including single sign-on, password recovery, etc.)
  2. Account configuration where users provide their company name and job role.
  3. Data upload: Upon uploading a file, users will receive a notification confirming receipt and a unique job ID which they will need to access their report later.

Build REST API and database on AWS

I created a REST API that enables communication between the front-end and our back-end. The main function of this API is to receive files uploaded by users and then upload them to an S3 bucket. The name of the folder will correspond to the name of the company, while the file name will be the unique job ID.

I also integrated the Bubble.io database with RDS using an SQL connector, to ensure that user data is updated accurately.
